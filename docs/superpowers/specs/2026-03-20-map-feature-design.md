# 地图功能设计

- 日期: 2026-03-20
- 主题: 将医院地图功能调整为前端数据驱动绘制，支持小程序地图 tab、导航页地图入口、扫码定位后的实时标注，以及 Vue 管理端地图预览与校验
- 状态: 已获用户确认，待按本设计实施

## 1. 背景与目标

当前系统已经具备以下基础能力：

- 数据库中存在规范化的医院节点与边数据
- 节点包含 `floor`、`x_coordinate`、`y_coordinate`
- 小程序已支持二维码扫码定位
- 小程序首页与导航页已维护当前位置、目的地与导航 session
- Vue 管理端已具备节点列表、二维码批量生成等运维入口

此前尝试基于静态 PNG 做地图展示，但该方案存在结构性问题：

- PNG 自带标题、图例、坐标轴与留白，数据库坐标难以稳定映射到图片像素
- 即使加入内框校准，不同导出图或不同屏幕上仍可能产生偏移
- 后续若增加路径高亮、设施筛选、节点点击等功能，图片方案扩展成本较高

因此本次地图功能改为：

- 小程序前端自行绘制楼层地图
- Vue 管理端前端自行绘制楼层地图
- 两端都直接复用数据库节点与边数据

本次目标是：

- 让用户在小程序中查看 1F/2F/3F 楼层地图
- 用户扫码后在地图中看到当前位置
- 用户选择目的地后在地图中看到目标点
- 用户导航时可在地图中看到下一校准点，并从导航页进入地图
- 管理端可用于核对节点、边线与坐标是否正确

## 2. 设计原则

### 2.1 数据为唯一真值

地图中的所有几何元素都由数据库数据驱动：

- 节点位置来自 `hospital_nodes`
- 边关系来自 `hospital_edges`
- 当前位置来自扫码定位与 `currentLocation`
- 目的地与下一校准点来自 `navigationSession`

前端不再依赖 PNG 像素坐标，也不维护第二套“图片定位坐标”。

### 2.2 渲染与数据分层

地图功能分为两层：

- 地图视图模型层
- 前端渲染层

视图模型层负责：

- 楼层过滤
- 坐标归一化
- 节点、边、marker、路径点组装

渲染层只负责把这些数据画出来。

### 2.3 按场景选择渲染技术

小程序：

- 使用 `canvas`
- 优先保证性能、兼容性与扫码导航联动

Vue 管理端：

- 使用 `SVG`
- 优先保证可读性、可调试性与交互便利性

### 2.4 第一版先保证结构准确与清晰可用

v1 不追求复刻 matplotlib 图片风格，重点是：

- 结构准确
- 点位准确
- 信息清晰
- 便于后续扩展

## 3. 范围

### 3.1 本次包含

- 小程序底部新增 `地图` tab
- 导航页保留“查看地图”入口
- 小程序地图页改为 canvas 绘制
- 小程序地图页支持 `1F / 2F / 3F` 切换
- 小程序地图页显示 `CURRENT / DESTINATION / SEGMENT_END`

- 连续室内定位
- 自动路线动画

## 4. 核心架构

### 4.1 地图视图模型

建议统一视图结构为：

#### `MapNodeView`

- `id`
- `nodeCode`
- `nodeName`
- `floor`
- `nodeType`
- `x`
- `y`
- `renderX`
- `renderY`

#### `MapEdgeView`

- `id`
- `fromNodeId`
- `toNodeId`
- `x1`
- `y1`
- `x2`
- `y2`

#### `MapMarkerView`

- `kind`: `CURRENT | DESTINATION | SEGMENT_END`
- `nodeId`
- `floor`
- `label`
- `renderX`
- `renderY`

#### `MapSceneView`

- `floor`
- `width`
- `height`
- `nodes`
- `edges`
- `markers`
- `routePoints`

### 4.2 坐标归一化

前端直接将数据库平面坐标映射到统一视口：

- `normalizedX = (x - minX) / (maxX - minX)`
- `normalizedY = (y - minY) / (maxY - minY)`
- `renderX = paddingLeft + normalizedX * drawableWidth`
- `renderY = paddingTop + (1 - normalizedY) * drawableHeight`

其中：

- `minX/maxX/minY/maxY` 在 v1 中继续沿用已经验证过的坐标范围
- 不同楼层共用同一范围，保证楼层切换时视觉尺度一致

### 4.3 数据来源

后端继续提供：

- 节点列表接口
- 边列表接口
- 单段导航接口

后端不参与绘图。

## 5. 小程序设计

### 5.1 页面结构

保留：

- `pages/map/map`

并继续作为底部 tab 页面。

### 5.2 页面布局

地图页包含：

- 顶部标题与模式提示
- 楼层切换按钮
- 中部 canvas 地图区域
- 底部状态摘要卡片

### 5.3 显示规则

未定位时：

- 显示楼层结构
- 不显示当前位置 marker

已定位时：

- 显示 `CURRENT`

已选目的地时：

- 显示 `DESTINATION`

导航中时：

- 显示 `SEGMENT_END`
- 若当前与目标楼层不同，提示优先前往楼梯或电梯换层

若导航 session 中存在当前段路径点，则在当前楼层高亮该路径折线。

### 5.4 不做项

- 地图自由拖拽
- 地图缩放
- 节点点击弹窗
- 3D 总览绘制

## 6. Vue 管理端设计

### 6.1 页面目标

管理端地图页用于：

- 查看楼层拓扑
- 核对节点分布
- 核对边线关系
- 点击节点查看详情

### 6.2 页面布局

保持三栏结构：

- 左侧筛选栏
- 中间 SVG 地图舞台
- 右侧节点详情

### 6.3 交互能力

支持：

- 楼层切换
- 节点名称搜索
- 节点类型筛选
- 边线开关
- 节点点击查看详情

## 7. 迁移策略

### 7.1 保留

- 扫码定位流程
- `currentLocation`
- `destination`
- `navigationSession`
- 节点与边接口
- 导航页“查看地图”入口

### 7.2 废弃

- 小程序地图页对 PNG 的运行时依赖
- Vue 地图页对 PNG 的运行时依赖
- 所有 `plotArea`、图片像素映射、图片加载错误兜底逻辑
- 3D 总览在运行时地图中的入口

### 7.3 实施顺序

1. 调整共享地图元数据与坐标归一化工具
2. 重构 Vue 地图页为 SVG 绘制
3. 重构小程序地图页为 canvas 绘制
4. 清理旧的 PNG 运行时代码

## 8. 风险与约束

- 小程序 canvas 需要正确处理容器尺寸与设备像素比，否则会出现模糊或拉伸
- 节点文字在局部密集区域可能重叠，v1 接受有限重叠
- 两端节点数据结构存在轻微差异，视图模型层需要兼容 `planarX/planarY` 与 `xCoordinate/yCoordinate`

## 9. 验收标准

- 小程序地图不再依赖 PNG 即可展示 1F/2F/3F 结构
- 扫码后当前位置能稳定落在对应节点上
- 选择目的地后目标点能稳定落在对应节点上
- 导航中能显示下一校准点，导航页入口仍可用
- Vue 管理端可按楼层预览节点和边线，并查看节点详情
